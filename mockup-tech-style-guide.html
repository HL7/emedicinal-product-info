Implementation Rules & Best Practices

Standard formatting and content structures for ePI narrative content — ensuring consistency, quality, and global interoperability across jurisdictions.

ℹ️ Scope This guide covers technical structure, XHTML compliance, and internal style attributes (text, tables, images). It does not include regulatory content requirements or external brand-specific styling.
⚠️ Important Do not include inline styling (fonts, colors, layouts) within the ePI document. Styling must remain in a separate stylesheet (CSS or XSLT). The only exceptions are basic inline formatting: bold, underline, italics, and text alignment.

Narrative Text — Allowed Elements
📝 Valid Top-Level Tags in ePI Narrative
<p> — Paragraph
Standard text blocks. Can contain <strong>, <em>, <sup>, <sub>, <img>. Do not nest lists or tables inside <p>.
<ol> / <ul> — Lists
Must only contain <li> children. List items may contain <div>, <p>, <img>, or nested lists.
<table> — Table
Use width="100%" for fluid layouts. Always include <thead> and <tbody>.
<footer> — Footer
Used for footnotes at the end of a section. May contain <p> and <a>.
<span> — Inline Span
For applying inline style classes to words or phrases without a block element.
Key Rules — Dos and Don'ts
✅ Formatting Rules
✅ Do
Use <strong> for bold
❌ Don't use <b>. The semantic <strong> tag improves accessibility.
✅ Do
Use <em> for italics
❌ Don't use <i>. Use <em> for semantic emphasis.
✅ Do
Use em/rem for font sizes
❌ Don't use fixed px font sizes — breaks mobile responsiveness.
✅ Do
Use width: 100% for tables
❌ Don't use fixed pixel widths — causes horizontal scrolling on mobile.
✅ Do
Use max-width: 100% for images
❌ Don't use fixed dimensions. Always include descriptive alt text.
✅ Do
Use UTF-8 character encoding
Ensures consistent cross-system text representation for all characters.

<ul> and <ol> tags must only contain <li> children. <li> items can contain <div>, <p>, headings, images, and tables as children.

Hyperlinks
🔗 Anchor Tag Usage

Use the <a> tag with its href attribute for hypertext links within ePI narrative. Example:

<a href="https://example.com">Visit site</a>

📌 Internal Section Links

Use anchor IDs for in-document cross-references between sections:

<a href="#section-4">See Section 4</a>

Tables

Only <thead>, <tbody>, <tfoot>, <caption>, and <colgroup> are permitted as direct children of <table>. Do not use tables for layout and alignment — use CSS alternatives instead.

<table>Defines a table
<caption>Table title (keep inside table)
<thead>Groups header rows
<tbody>Groups body rows
<tfoot>Groups footer rows (keep footnotes inside table)
<tr>Defines a row
<th>Defines a header cell
<td>Defines a data cell
<colgroup> / <col>Column grouping & formatting
colspan / rowspanMerge cells across columns / rows

Images
🖼️ Preferred Format: SVG

SVG (Scalable Vector Graphics) is preferred for ePI images due to lossless scalability across devices. JPEG and PNG are acceptable for legacy content only.

📦 Embedding: Base64 Binary

All images must be embedded as Base64 objects within a Contained Binary resource to ensure the ePI is exchanged as a single self-contained file.

♿ Accessibility: alt Text

Always include descriptive alt text for every image. Ensure colors within images meet WCAG Level AA contrast requirements.

📱 Responsiveness

Use max-width: 100%; height: auto; for all images to ensure they scale correctly across desktop and mobile viewports.

Audio & Video
ℹ️ Best Practice Do not embed audio or video files directly within an ePI — their large size makes the document impractical to exchange. Instead, link to external copies using the <a> tag. Always check local regulations to confirm if audio and video are permitted in your jurisdiction.
Languages
🌍 One Language per ePI

Each language must be a separate ePI document. Do not mix multiple languages within a single ePI Composition.

↔️ Text Direction

Text direction (e.g., RTL for Arabic, vertical for Japanese) is managed by the stylesheet — not the underlying XML/JSON content.

Math Formulas
∑ MathML is not currently supported in FHIR ePI
Simple Formulas
Recreate using inline narrative tags. Example: a + b<sup>2</sup> renders as a + b²
Complex Formulas
Embed as images (SVG preferred) using the Contained Binary approach described in the Images section above.
